# 🧪 멍빗어 통합 테스트 가이드

## 📝 테스트 개요

이 문서는 멍빗어 애플리케이션의 통합 테스트 시나리오를 설명합니다.

---

## 1️⃣ **회원 관리 테스트**

### 1.1 회원가입 테스트
```
시나리오: 새로운 사용자가 회원가입합니다
전제조건: 
  - 앱이 정상 로드됨
  - Firebase Authentication 활성화됨

단계:
  1. 로그인 페이지에서 "회원가입" 버튼 클릭
  2. 이메일 입력 (예: test@example.com)
  3. 비밀번호 입력 (예: TestPassword123!)
  4. 이름 입력 (예: 테스트 사용자)
  5. "회원가입" 버튼 클릭
  6. 대시보드로 이동 확인

예상 결과: 
  ✅ 새 사용자 계정 생성됨
  ✅ Firestore의 users 컬렉션에 데이터 저장됨
  ✅ 자동으로 로그인되어 대시보드 이동
```

### 1.2 로그인 테스트
```
시나리오: 기존 사용자가 로그인합니다

단계:
  1. 로그인 페이지에서 이메일 입력
  2. 비밀번호 입력
  3. "로그인" 버튼 클릭
  4. 대시보드 이동 확인

예상 결과:
  ✅ 로그인 성공
  ✅ 사용자 프로필 정보 로드됨
  ✅ 대시보드에서 사용자 강아지 목록 표시
```

### 1.3 로그아웃 테스트
```
시나리오: 사용자가 로그아웃합니다

단계:
  1. 마이페이지 > 계정 관리
  2. "로그아웃" 버튼 클릭
  3. 로그인 페이지로 이동 확인

예상 결과:
  ✅ 로그아웃 성공
  ✅ 로그인 페이지 표시
  ✅ 이전 사용자 데이터 캐시는 제거됨
```

## 2️⃣ **강아지 관리 테스트**

### 2.1 강아지 등록 테스트
```
시나리오: 사용자가 새로운 강아지를 등록합니다

단계:
  1. 대시보드에서 "강아지 등록" 버튼 클릭
  2. 강아지 이름 입력 (예: 뽀삐)
  3. 품종 선택 (예: 푸들)
  4. 나이 입력 (예: 3)
  5. 체중 입력 (예: 5.5)
  6. (선택) 사진 업로드
  7. "등록" 버튼 클릭

예상 결과:
  ✅ 강아지 정보 등록됨
  ✅ Firestore의 users/{userId}/dogs 컬렉션에 저장됨
  ✅ 이미지 업로드 시 Firebase Storage에 저장됨
  ✅ 대시보드의 강아지 목록에 표시됨
```

### 2.2 강아지 정보 수정 테스트
```
시나리오: 사용자가 강아지 정보를 수정합니다

단계:
  1. 마이페이지 > 내 강아지
  2. 수정할 강아지 선택
  3. 정보 수정 (예: 나이 변경)
  4. "저장" 버튼 클릭

예상 결과:
  ✅ 강아지 정보 수정됨
  ✅ Firestore에 업데이트됨
  ✅ 변경사항이 즉시 반영됨
```

### 2.3 강아지 삭제 테스트
```
시나리오: 사용자가 강아지를 삭제합니다

단계:
  1. 강아지 관리 페이지에서 삭제할 강아지 선택
  2. "삭제" 버튼 클릭
  3. 확인 메시지 수락

예상 결과:
  ✅ 강아지 삭제됨
  ✅ Firestore에서 문서 삭제됨
  ✅ 목록에서 제거됨
```

## 3️⃣ **디자이너 검색 테스트**

### 3.1 디자이너 목록 조회 테스트
```
시나리오: 사용자가 디자이너 목록을 조회합니다

단계:
  1. 대시보드에서 "디자이너 찾기" 클릭
  2. 디자이너 목록 로드 확인

예상 결과:
  ✅ Firestore의 designers 컬렉션에서 모든 디자이너 로드됨
  ✅ 디자이너 카드가 평점 순으로 정렬됨
  ✅ 디자이너 정보 (이름, 가격, 평점) 표시됨
```

### 3.2 디자이너 검색 및 필터 테스트
```
시나리오: 사용자가 디자이너를 검색합니다

단계:
  1. 검색 페이지 이동
  2. 위치 입력 (예: 서울)
  3. 가격대 선택 (예: 30,000~50,000)
  4. "검색" 버튼 클릭

예상 결과:
  ✅ 조건에 맞는 디자이너만 필터링됨
  ✅ 결과 수 표시
  ✅ 정렬 옵션으로 재정렬 가능
```

### 3.3 디자이너 상세 정보 조회 테스트
```
시나리오: 사용자가 디자이너 상세 정보를 확인합니다

단계:
  1. 디자이너 목록에서 디자이너 선택
  2. 상세 페이지 로드

예상 결과:
  ✅ 디자이너 프로필 정보 로드 (이름, 경력, 설명)
  ✅ 포트폴리오 이미지 표시
  ✅ 최근 후기 3-5개 표시
  ✅ 별점 평균 계산 및 표시
```

## 4️⃣ **예약 및 결제 테스트**

### 4.1 견적 요청 테스트
```
시나리오: 사용자가 미용 견적을 요청합니다

단계:
  1. 디자이너 상세 페이지에서 "견적 요청" 클릭
  2. 페이지별로 정보 입력:
     - Step 0: 강아지 선택
     - Step 1: 미용 타입 선택 (목욕, 미용 등)
     - Step 2: 특별 요청사항 입력
     - Step 3: 선호 기간 선택
     - Step 4: 추가 질문 입력
  3. "요청 제출" 버튼 클릭

예상 결과:
  ✅ 견적 요청이 Firestore의 quotes 컬렉션에 저장됨
  ✅ 디자이너가 알림 수신
  ✅ 요청 성공 페이지 表示
```

### 4.2 예약 테스트
```
시나리오: 사용자가 미용 일정을 예약합니다

단계:
  1. 디자이너 선택 후 "즉시 예약" 클릭
  2. 캘린더에서 날짜 선택
  3. 시간 슬롯 선택 (예: 10:00 ~ 12:00)
  4. 강아지 선택
  5. "다음" 버튼 클릭

예상 결과:
  ✅ 선택한 날짜/시간이 유효함
  ✅ 예약 정보가 다음 페이지로 전달됨
  ✅ 캘린더에 예약된 시간이 비활성화됨
```

### 4.3 결제 테스트
```
시나리오: 사용자가 예약을 결제합니다

단계:
  1. 결제 페이지에서 예약 정보 확인
  2. 신용카드 정보 입력:
     - 카드 번호 (예: 1234567890123456)
     - 만료일 (예: 12/25)
     - CVV (예: 123)
  3. 이용약관 동의 체크
  4. "결제" 버튼 클릭

예상 결과:
  ✅ 결제 유효성 검사 성공
  ✅ 예약이 Firestore의 bookings 컬렉션에 저장됨
  ✅ 결제 확인 페이지 표시
  ✅ 사용자와 디자이너가 알림 수신
```

## 5️⃣ **실시간 기능 테스트**

### 5.1 채팅 테스트
```
시나리오: 사용자와 디자이너가 실시간 메시지를 주고받습니다

단계:
  1. 채팅 페이지 이동
  2. 디자이너와의 채팅방 선택
  3. 메시지 입력 (예: "안녕하세요")
  4. 전송 버튼 클릭
  5. 메시지가 화면에 표시되는지 확인
  6. (다른 브라우저/탭에서) 메시지 수신 확인

예상 결과:
  ✅ 메시지가 Firestore에 저장됨
  ✅ 메시지가 실시간으로 표시 (onSnapshot 리스너)
  ✅ 메시지에 전송 시간 표시됨
  ✅ 읽음/미읽음 상태 표시 (✓/✓✓)
  ✅ 자동 스크롤로 최신 메시지 표시
```

### 5.2 알림 테스트
```
시나리오: 사용자가 실시간 알림을 받습니다

단계:
  1. 알림 페이지 이동
  2. 다른 윈도우에서 알림 생성 이벤트 트리거:
     - 예: 새 메시지 도착, 예약 확정 등
  3. 알림 페이지에서 자동 업데이트 확인

예상 결과:
  ✅ 알림이 Firestore에 저장됨
  ✅ 알림이 실시간으로 표시 (onSnapshot 리스너)
  ✅ 미읽음 알림은 진하게 표시
  ✅ 알림 클릭 시 읽음 처리
  ✅ 알림 종류별 아이콘 표시 (💬, 📅, ⭐ 등)
```

## 6️⃣ **이미지 업로드 테스트**

### 6.1 강아지 사진 업로드 테스트
```
시나리오: 사용자가 강아지 사진을 업로드합니다

단계:
  1. 강아지 등록 페이지에서 사진 선택
  2. JPG/PNG 파일 선택 (크기 < 5MB)
  3. "등록" 버튼 클릭

예상 결과:
  ✅ Firebase Storage의 users/{userId}/dogs/{dogId}에 저장됨
  ✅ 다운로드 URL이 Firestore에 저장됨
  ✅ 사진이 프로필에 표시됨
  ✅ 용량 초과 시 에러 메시지 표시
```

### 6.2 후기 사진 업로드 테스트
```
시나리오: 사용자가 후기에 사진을 첨부합니다

단계:
  1. 후기 작성 페이지에서 사진 선택
  2. 이미지 업로드
  3. "등록" 버튼 클릭

예상 결과:
  ✅ Firebase Storage의 users/{userId}/reviews/{reviewId}에 저장됨
  ✅ 후기와 사진이 함께 저장됨
  ✅ 디자이너 상세 페이지에서 사진 표시
```

## 7️⃣ **성능 테스트**

### 7.1 로딩 속도 테스트
```
시나리오: 각 페이지의 로딩 시간 측정

측정 대상:
  - 초기 로드 시간: < 3초
  - 페이지 전환: < 1초
  - 이미지 로드: < 2초
  - API 응답: < 1초

도구:
  - Chrome DevTools > Lighthouse
  - Chrome DevTools > Network 탭
```

### 7.2 네트워크 느림 상태 테스트
```
시나리오: 느린 네트워크 환경에서 앱 테스트

단계:
  1. Chrome DevTools > Network 탭
  2. Throttling 설정: "Slow 3G"
  3. 주요 기능 테스트

예상 결과:
  ✅ 로딩 인디케이터 표시
  ✅ 타임아웃 처리
  ✅ 오류 메시지 표시
```
